经过一系列的分析、代码注入和诊断，我们成功破解并复现了完整的签名流程。以下是最终确认的核心要素：

*   **API 端点:** `https://api.cherry-ai.com/chat/completions`
*   **HTTP 方法:** `POST`
*   **签名算法:** `HMAC-SHA256`
*   **最终密钥 (Secret Key):**
    ```
    K3RNPFx19hPh1AHr5E1wBEFfi4uYUjoCFuzjDzvS9cAWD8KuKJR8FOClwUpGqRRX.GvI6I5ZrEHcGOWjO5AKhJKGmnwwGfM62XKpWqkjhvzRU2NZIinM77aTGIqhqys0g
    ```
*   **签名载荷 (Signing Payload) 结构:**
    一个由**六个部分**组成的字符串，以换行符 `\n` 分隔，结构如下：
    ```
    POST
    /chat/completions
    [一个空字符串]
    cherry-studio
    [Unix 时间戳 (秒)]
    [未经排序的原始 JSON 请求体字符串]
    ```
*   **签名后处理:**
    计算出的 HMAC-SHA256 十六进制摘要**无需任何额外处理**（如反转）。
*   **必要的 HTTP Headers:**
    *   `x-signature`: 计算出的签名。
    *   `x-timestamp`: 用于签名的 Unix 时间戳。
    *   `x-client-id`: 固定值为 `cherry-studio`。
    *   `x-title`: 固定值为 `Cherry Studio`。

## 5. 最终 Python 实现

以下是经过千锤百炼后，最终成功的 `generate_signature` 函数实现：

```python
import json
import time
import hmac
import hashlib
from typing import Dict, Any

# 最终密钥
HMAC_SECRET_KEY = "K3RNPFx19hPh1AHr5E1wBEFfi4uYUjoCFuzjDzvS9cAWD8KuKJR8FOClwUpGqRRX.GvI6I5ZrEHcGOWjO5AKhJKGmnwwGfM62XKpWqkjhvzRU2NZIinM77aTGIqhqys0g"

def generate_signature(request_body: Dict[str, Any]) -> Dict[str, str]:
    """
    根据最终逆向结果，在 Python 中 1:1 复现 Cherry Studio 的签名算法。
    """
    method = "POST"
    path = "/chat/completions"
    query = ""
    client_id = "cherry-studio"
    timestamp = str(int(time.time()))
    
    # 关键：不排序，保持客户端发送的原始 JSON 键顺序
    body_str = json.dumps(request_body, separators=(',', ':'), ensure_ascii=False)
    
    # 构建与原始程序完全一致的签名载荷
    signing_payload = f"{method}\n{path}\n{query}\n{client_id}\n{timestamp}\n{body_str}"
    
    # 计算 HMAC-SHA256 签名
    signature = hmac.new(
        HMAC_SECRET_KEY.encode('utf-8'),
        signing_payload.encode('utf-8'),
        hashlib.sha256
    ).hexdigest()

    # 返回所有必要的请求头
    return {
        "x-timestamp": timestamp,
        "x-signature": signature,
        "x-client-id": client_id,
        "x-title": "Cherry Studio"
    }
```
